2.1 Aprior算法

背景介绍

​ Apriori算法是常⽤的⽤于挖掘出数据关联规则的算法，它⽤来找出数据值中频繁出现的数据集合，找出这些集合的模式有助于我们做⼀些决策。频繁集：出现次数多的数据集。

频繁项集的评估规则

  • 支持度：几个关联的数据在数据集中出现的次数占总数据集的比重；联合概率。

    例：用户中购买啤酒和尿布的占总体的比例。

    注：支持度高的数据不一定构成频繁项集，但是支持度太低的数据肯定不构成频繁项集。

  • 置信度：一个数据出现后，另一个数据出现的概率；条件概率。

    例：在购买啤酒的用户中，购买了尿布的比例。

  • 提升度：含有Y的条件下，同时含有X的概率，与X总体发生的概率之比。

算法思想

  • 对于Apriori算法，我们使⽤⽀持度来作为我们判断频繁项集的标准。Apriori算法的⽬标是找到最⼤的K项频繁集。
  • 找到符合⽀持度标准的频繁集，因为这样的频繁集可能有很多，所以我们要找到最⼤个数的频繁集。
  • 采⽤了迭代的⽅法，先搜索出候选1项集及对应的⽀持度，剪枝去掉低于⽀持度的1项集，得到频繁1项；然后产生2项集，剪去低于支持度的2项….直至剩下的k项集只有一个为止，则返回结果。

算法流程

  • 输⼊：数据集合D，⽀持度阈值$\alpha$

  • 输出：最⼤的频繁k项集　　

  • 扫描整个数据集，得到所有出现过的数据，作为候选频繁1项集。k=1，频繁0项集为空集。

  • 挖掘频繁k项集

    • 扫描数据计算候选频繁k项集的⽀持度
    • 去除候选频繁k项集中⽀持度低于阈值的数据集,得到频繁k项集。如果得到的频繁k项集为空，则直接返回频繁k-1项集的集合作为算法结果，算法结束。如果得到的频繁k项集只有⼀项，则直接返回频繁k项集的集合作为算法结果，算法结束。
    • 基于频繁k项集，连接⽣成候选频繁k+1项集。

  • 令k=k+1，转⼊上一步。

​ 注：从算法的步骤可以看出，Aprior算法每轮迭代都要扫描数据集，因此在数据集很⼤，数据种类很多的时候，算法效率很低

2.2 FP Growth算法（别名：FP Tree算法）

基本介绍

​ 相比于Apriori算法需要多次扫描数据，I/O是很大的瓶颈。为了解决这个问题，FP Growth算法采用了一些技巧，任何情况下仅需要扫描两次数据集，因此提高了算法运行的效率

​ 挖掘强关联规则就是在满足一定支持度的情况下，寻找置信度达到阈值的所有商品的组合。

FP Tree 数据结构

项头表：记录了所有的1项频繁集出现的次数，按照次数降序排列，比如B在原始数据中出现8次。

节点链表：所有项头表里的1项频繁集都是一个节点链表的头，它依次指向FP树中该1项频繁集出现的位置，这样方便项头表和FP Tree之间的联系查找和更新。

FP树：将原始数据映射到内存中的一颗FP树。

项头表的建立

第一次扫描原始数据：得到所有1项频繁集的计数，删除支持度低于阈值（这里为20%）的项，将1项频繁集放入项头表，并按照支持度降序排列。目的：建立项头表

第二次扫描原始数据：将原始数据剔除支持度低于阈值的项，并按照支持度降序排列。目的：筛选原始数据

解释

  • 上图中，第一次扫描原始数据，因为O，I，L，J，P，M, N都只出现一次，而支持度为2（$10 \times 20 \% = 2$），所以在建立项头表的时候，剔除上述字母，并将其他字母出现次数按照降序的方式排列；
  • 第二次扫描原始数据，将数据项中出现的上述被剔除的字母（O，I，L，J，P，M, N），在原始数据项中剔除。

FP Tree的建立

  1. 将排序后的数据集按照顺序插入FP树，排序靠前的节点是祖先节点，而靠后的是子孙节点。
  2. 如果有共用的祖先，则对应的公用祖先节点计数加1。
  3. 如果有新节点出现，则项头表对应的节点会通过节点链表链接上新节点。
  4. 直到所有的数据都插入到FP树后，FP树的建立完成。

FP Tree的挖掘

  1. 首先要从项头表的底部项依次向上挖掘，对于项头表对应于FP树的每一项，我们要找到它的条件模式基。所谓条件模式基是以我们要挖掘的节点作为叶子节点所对应的FP子树；

  2. 得到FP子树后，我们将子树中每个节点的的计数设置为叶子节点的计数，并删除计数低于支持度的节点；

  3. 从这个条件模式基，我们就可以递归挖掘得到频繁项集；

    图1:

    举例（已知F的最大频繁项已经找到，我们这里举例D）

    • 从项头表的最底部开始，选取D，找到具有D叶子节点的FP子树；条件基分别为{A,C,D,D}和{A,C,E,G,D}。（如图2）
    • 将所有的祖先节点计数设置为叶子节点的计数，即为{A:2, C:2,E:1 G:1,D:1, D:1}（A和C为两个D的祖先节点，所以计数为2）。此时E节点和G节点由于在条件模式基里面的支持度（这里为2）低于阈值，被我们删除，最终在去除低支持度节点并不包括叶子节点后D的条件模式基为{A:2, C:2}；
    • 所以，D的频繁2项集为{A:2,D:2}, {C:2,D:2}；
    • 递归合并二项集，得到频繁三项集为{A:2,C:2,D:2}。
    • 此例中D的最大频繁集为频繁三项集

FP Tree的算法流程

  1. 扫描数据，得到所有频繁一项集的的计数。然后删除支持度低于阈值的项，将1项频繁集放入项头表，并按照支持度降序排列。
  2. 扫描数据，将读到的原始数据剔除非频繁1项集，并按照支持度降序排列。
  3. 读入排序后的数据集，插入FP树，插入时按照排序后的顺序，插入FP树中，排序靠前的节点是祖先节点，而靠后的是子孙节点。如果有共用的祖先，则对应的公用祖先节点计数加1。插入后，如果有新节点出现，则项头表对应的节点会通过节点链表链接上新节点。直到所有的数据都插入到FP树后，FP树的建立完成。
  4. 从项头表的底部项依次向上找到项头表项对应的条件模式基。从条件模式基递归挖掘得到项头表项项的频繁项集。
  5. 如果不限制频繁项集的项数，则返回步骤4所有的频繁项集，否则只返回满足项数要求的频繁项集。
