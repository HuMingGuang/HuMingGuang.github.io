一、基本语义问题

1.变量和对象

序号 c语言 Python
1 值语义 引用语义
2 变量必须声明类型，以便安排存储，不同类型的变量大小不同，也导致一种类型的变量不能保存其他类型的值。 变量只需保存对象标识，所有变量大小相同。变量不需要说明，赋值即定义。变量没有类型，一个变量完全可以一会关联整数，一会关联字符串。
3 缺点：由于变量无类型，解释器不能根据代码中变量确定操作是否正常进行，必须在运行中检查。运行中频繁检查时Python程序效率低的一个重要原因。

在Python中数据值是独立对象，赋给变量就是建立关联，以便通过变量使用对象。

三、生成器函数和必包

1.生成器函数

定义生成器函数

定义生成器函数的语法形式与普通函数形式一样，特殊之处就是函数体出现了yield语句。

yield语句基本形式: yield 表达式

调用生成器函数得到一个生成器对象，这是一种迭代器，可以用在需要迭代器的上下文中(比如：for语句)，上下文向它要求一个值时，该对象就会执行函数体代码，直到遇到yield语句时送出表达式的值。注意：这时生成器并没有结束，而是停在该yield语句处，再次被要求值时，它就从暂停位置继续送出下一个值。

由生成器函数产生的生成器对象能保持自己的执行状态，并一次次的送出值。

2.闭包技术和原理

闭包技术并不是一种特殊的Python语言结构，而是一种非常有用的编程技术。从一个函数返回局部函数的技术，称为闭包技术，也可称之为工厂函数。

闭包技术的基本结构：局部函数，局部变量，return 局部函数

带接口的闭包

闭包函数f返回一个局部函数，该函数可以访问和修改f调用时建立的名字空间，因此可以看作f生成闭包的接口函数。闭包包装了一些数据（内部状态），通过接口函数提供对数据的操作，可以构成任意复杂的数据抽象。

四、异常和异常处理

1.处理错误的基本原则

  • 程序运行中遇到错误情况，应尽可能以某种方式继续执行（不随意终止执行，不崩溃），较少损失
  • 即使无法处理错误，也要尽可能保存信息，不出现非预期行为
  • 在一个局部发生的错误应尽可能局部处理

2.防御性程序设计

在适当的位置，加入检查代码，让各代码单元尽可能保护自己，只在正确状态下做正确的事情。

3.异常处理和try结构

try结构专用于描述异常的捕捉和处理，包括发生异常后的处理操作和流程。

try结构包装可能发生异常的代码段，用try结构后部的except段描述执行中发生异常的处理操作。头部为exacpt:时，表示捕捉一切异常。
