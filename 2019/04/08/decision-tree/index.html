1. 决策树基础知识

1.1 定义

决策树是一种树状模型，可以进行基本的分类与回归，另外它也是进行集成方法经常用到的基模型。决策树模型，一般分为三部分：特征选择、决策树的生成和决策树的剪枝。

2. 几种常用的决策树算法

2.1 ID3-最大信息增益

ID3 算法的核心是在决策树各个结点上应用信息增益准则选择特征，递归地构建决策树。

具体方法

  • 从根结点开始，对结点计算所有可能的特征的信息增益；
  • 选择信息增益最大的特征作为结点的特征，由该特征的不同取值建立子结点；
  • 再对子结点递归地调用以上方法，构建决策树；
  • 直到所有特征的信息增益均很小或没有特征可以选择为止，最后得到一个决策树。

2.2 C4.5-最大信息增益比

ID3决策树存在三个缺点

  • 使用信息增益来选择特征，偏向于优先选取取值较多的特征
  • 不能处理连续值特征
  • 容易过拟合

针对上述三个问题，C4.5决策树给出了解决方法

问题一：ID3决策树容易偏向于优先选取取值较多的特征

解决方法：用信息增益比代替信息增益

问题二：ID3决策树不能解决连续值特征

解决方法：将连续的数值特征进行从小到大排序，得到属性值集合,将区间的中位数作为候选划分点，于是得到包含个元素的划分点集合.基于每个划分点，我们可以将样本分为两类，从而可以得到基于划分点的信息增益值，选取信息增益值最大的划分点，对样本进行划分。

问题三：ID3决策树容易过拟合

解决方法：决策树过拟合问题主要是由于树的分叉过细造成的，C4.5引入了正则化系数进行初步的剪枝来缓解过拟合问题。

2.3 CART-最小基尼指数(Gini)

2.3.1 ID3树和C4.5树的改进——CART树

C4.5决策树使用熵模型，存在大量的对数计算，计算过程较为复杂；其次ID3决策树和C4.5决策树采用多叉树形式，一旦使用其特征划分，之后不能再利用，特征信息的利用率低；C4.5决策树对连续数值的处理方式是按区间将其离散化，这样会损失该特征的一部分内部信息。

问题一：ID3决策树和C4.5决策树特征选择过程对数计算过于复杂

CART树在分类任务时，使用基尼指数作为特征选择的标准；CART树在回归任务时，采用平方误差最小化进行特征选择，这两种方式都减少了大量的对数运算。

问题二：ID3决策树和C4.5决策树对特征划分过于迅速

ID3决策树和C4.5决策树采用多叉树进行特征划分，CART分类树和回归树采用二叉树对每个特征进行划分。值的注意的是ID3决策树和C4.5决策树每次只需计算该特征的加入带来的信息增益比；但是CART树采用二叉树划分，实际在每次特征选择时计算了某个特征下某个划分点的基尼系数或平方误差。即前者每次进行特征选择的最小单位是某个特征；而后者最小单位是某个特征下的某个切分点。这样CART树可以对同一个特征进行多次利用。

问题三：ID3决策树和C4.5决策树不能处理回归问题

使用均方误差的方式，代替信息增益或者信息增益比；同时在预测回归时，采用叶子节点中所有样本点标签值的均值。

3. 决策树剪枝

一棵完全生长的决策树面临一个很严重的问题，即过拟合。我们需要对决策树进行剪枝，剪掉一些枝叶，提升模型的泛化能力。

3.1 预剪枝(Pre-Pruning)

在生成决策树的过程中，停止树的增长。

核心思想：在树中结点进行扩展之前，先计算当前的划分是否能带来模型泛化能力的提升，如果不能，则不再继续生长子树。此时可能存在不同类别的样本同时存于结点中，按照多数投票的原则判断该结点所属类别。

何时停止决策树的生长

  • 当树到达一定深度时，停止树的生长
  • 当到达当前结点的样本数量小于某个阈值时，停止树的生长
  • 计算每次分裂对测试集的准确度提升，当小于某个阈值时，不再继续扩展

预剪枝存在一定的局限性，有欠拟合的风险，虽然当前的划分导致测试集准确率降低，但是在之后的划分中，准确率有显著提升。

3.2 后剪枝(Post-Pruning)

在已生成的过拟合决策树上进行剪枝，得到简化版的剪枝决策树。

核心思想：让算法生成一棵完全生长的决策树，然后从底层向上计算是否剪枝。剪枝过程将子树删除，用一个叶子结点替代，该结点的类别同样按照多数投票的原则进行判断。剪枝过程，如果在测试集上的准确率有所提高，则进行剪枝。

相比于预剪枝，后剪枝通常得到的决策树泛化能力更强，但是时间开销会更大。

4. 小结

4.1 算法比较

输入数据类型 树类型 特征选择标准 输出类型
ID3 类别 二叉树/多叉树 信息增益 二分类/多分类
C4.5 类别、数值 二叉树/多叉树 信息增益比 二分类/多分类
CART树分类 类别、数值 只能二叉树 基尼指数 二分类/多分类
CART树回归 类别、数值 只能二叉树 平方误差 回归

4.2 优缺点

4.2.1 优点

  • 决策树模型可视化，简单易理解
  • 可处理类别型数据、数值型数据
  • 可处理多分类和非线性分类问题
  • 可作为集成模型的基模型

4.2.2 缺点

  • 决策树模型对噪声比较敏感，噪声较大，容易产生过拟合
  • 处理特征关联性较强的数据时，表现不太好
