一、装饰器

1.装饰器

定义：不改变程序部件基本行为而增加某种附加功能的技术；例如@staticmethod

和@classmethod都是标准的装饰器

2.用函数实现函数装饰器

用函数装饰函数时，涉及到两个函数：一个是被装饰函数，一个是实施装饰函数（又称为装饰器函数），我们的目标是不修改被装饰函数定义的情况下，通过装饰器函数给它增加一些功能，装饰器的返回值也是一个函数对象。

3.用类作为函数装饰器

一个装饰器就是一个可调用对象，它也可调用对象为参数，返回可调用对象。类也是可调用对象，也可以用于定义函数装饰器。基本思想：用类实例模拟装饰后的函数，创建实例时以被装饰函数作为参数。类中定义了一个__call__()方法，实现装饰后的函数的功能。

4.2日志文件

在实际系统的执行或测试中，经常需要把一组函数的调用轨迹（或其他信息）记入一个文件，这种文件称为日志文件或log文件。

二、异步程序和协程

1.threading 和 multiprocessing

threading包支持基于线程的并发编程，可用于创建能并发执行的线程，启动和控制其执行，实现线程间的信息交换和合作。另一个包multiprocessing支持基于进程的并发，支持进程的分裂、合并等操作，并能支持真正的并发执行。

2.协程的优点和缺点

  • 优点
    • 每个执行单元的额外存储开销小
    • 协程之间切换的代价非常低
  • 缺点
    • 如果一个协程长期霸占控制权，不主动挂起，就会导致其他协程得不到执行，影响整个系统的正常运行
